# KCitychatBot: A knowledge graph based chatbot system for large scale CityGML dataset
A chatbot based on llm and kg to translate citygml into natural language.
---
This repository is the official implementation of [<u>"KCitychatBot: A knowledge graph based chatbot system for large scale CityGML dataset"</u>](https://www.csis.u-tokyo.ac.jp/3d_geoinfo_sdsc_2025/overview.html)

## 1.Abstract
CityGML has been extensively studied for its ubiquitous existence in numerous domains. However, its complicated hierarchical structure still poses difficulties in large-scale storage and usability for non-expert users. Recently, large language model(LLMs) have exhibited significant strengths in natural language processing(NLP) and dialogue systems. Nevertheless, LLMs are highly dependent on pre-trained data, which can give rise to hallucination problems and context length constraints. To address these issues, we firstly propose a novel method for transforming CityGML data into knowledge graphs by leveraging a graph database and a Python driver. This approach effectively mitigates the challenges of storing and representing the complex structure of CityGML and can be applied as an external knowledge base for the dialogue system. Secondly, we develop a collaborative multi-agent framework to perform natural language queries over CityGML data in a human-friendly manner. By integrating the constructed knowledge graphs with several knowledge augmentation strategies, this chatbot system accomplishes a complete pipeline from natural language input to structured query generation, external knowledge retrieval, and optimized response generation. To evaluate the accuracy and interpretability of the system's outputs, we conduct experiments on both the construction of the city knowledge graphs and the design of the collaborative multi-agent framework. The experimental results demonstrate that the system achieves expected levels of performance in terms of answer accuracy, relevance, and contextual coherence. These findings underscore its potential to lower the barrier to entry for non-professionals interacting with CityGML data, providing theoretical insights as well as practical implications for advancing CityGML applications and promoting smart city development.
